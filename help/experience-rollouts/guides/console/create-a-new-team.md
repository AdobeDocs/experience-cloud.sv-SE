# Skapa ett nytt team {#create-a-new-team}

Om det inte finns något team för ditt projekt i Experience Rollouts måste du begära ett innan du kan börja skapa funktionsflaggor. Teamskapande hanteras via en supportförfrågan.

## Innan du börjar {#before-you-begin}

Bekräfta att det inte redan finns ett team för ditt projekt. Kontrollera med din produkt eller tekniker eller granska [Team och deras administratörer](teams-and-admins.md). Om ett team redan finns ber vi dig att gå med i det istället - se [Begär åtkomst](request-access.md).

## Skicka en teamförfrågan {#submit-request}

Kontakta supporten för Experience Rollouts för att begära ett nytt team. Ange följande information i din begäran:

| Fält | Beskrivning |
|---|---|
| **Teamnamn** | Ett unikt namn för ditt team. |
| **Målläge** | Målgruppsläget som teamet använder: [målgruppsanpassning på användarnivå eller målinriktning på organisation- och miljönivå](../../getting-started/experience-rollouts-modes.md). |
| **Miljö** | Den eller de miljöer du behöver: Stage, Production eller båda. |
| **Teamadministratörer** | E-postadress(er) till den eller de personer som ska fungera som gruppadministratörer. |

>[!NOTE]
>
>Teamadministratörer ansvarar för att hantera medlemmar och godkänna åtkomstbegäranden. Utse minst två administratörer för att undvika åtkomstluckor om en administratör inte är tillgänglig.

## När teamet har skapats {#after-creation}

När teamet har skapats kan administratörer börja lägga till medlemmar på ett självbetjäningssätt. Nya medlemmar begär åtkomst via åtkomsthanteringssystemet och administratörer godkänner eller avvisar dessa begäranden.

Se [Begär åtkomst](request-access.md) för de steg som medlemmarna ska följa för att gå med i teamet.
